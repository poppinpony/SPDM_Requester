# Claude Code Skills 參考

本專案可用的 Claude Code 技能（使用 `/skill-name` 呼叫）。

## `/simplify`

審查最近修改的程式碼，檢查可重用性、品質與效率，並修正發現的問題。

**適合用在：**
- 剛寫完一段新功能，想做 code review
- 重構後確認品質
- 懷疑有重複邏輯

---

## `/loop [interval] [command]`

以固定間隔重複執行指令。預設間隔 10 分鐘。

**範例：**
```
/loop 5m /simplify      ← 每 5 分鐘執行一次 simplify
```

**適合用在：**
- 輪詢背景測試結果
- 定期確認 build 狀態

---

## `/keybindings-help`

客製化 Claude Code 快捷鍵（修改 `~/.claude/keybindings.json`）。

**適合用在：**
- 綁定常用操作到快捷鍵
- 修改 submit 鍵

---

## `/claude-api`

建立使用 Claude API / Anthropic SDK 的應用程式。

**觸發時機：** 程式碼 import `anthropic`、`@anthropic-ai/sdk`、或 `claude_agent_sdk`

**本專案暫不使用**（工具層為 Python ctypes + C，不依賴 Anthropic SDK）

## Agent 行為準則（agent-principles）

AI 協作開發的核心行為規範。開始新功能、審查程式碼品質、或輸出看起來過於複雜時套用。

---

### 1. 先想清楚，再動手

**不假設、不掩蓋困惑、主動呈現取捨。**

#### 應該做
- 明確說出假設——不確定時，問清楚再動，不要猜
- 存在多種解讀時，全部列出，不要默默選一個
- 有更簡單的做法時，主動說出來
- 遇到困惑就停下來，說明哪裡不清楚，請使用者確認

#### 不應該做
- 默默假設然後埋頭做
- 用自信的語氣掩蓋不確定性
- 需求模糊時還繼續往前推進

#### 自我檢查
每一個決策，都能追溯到明確的需求或使用者的確認嗎？

---

### 2. 簡單優先

**解決問題的最少程式碼。不做任何推測性的設計。**

#### 應該做
- 先實作最直觀的解法
- 超過 100 行就反問自己有沒有必要
- 能把 200 行寫成 50 行就重寫

#### 不應該做
- 加入使用者沒要求的功能
- 為只用一次的邏輯建立抽象
- 加入沒被要求的「彈性」或「可設定性」
- 為不可能發生的情境加 error handling
- 以「以後可能會用到」為由過度設計

#### 自我檢查
資深工程師會覺得這段程式碼過度複雜嗎？如果會，就簡化。

---

### 3. 精準修改

**只動必須動的地方。只清理自己製造的問題。**

#### 修改既有程式碼時
- 不「順便優化」旁邊的程式碼、註解或格式
- 不重構沒壞的東西
- 維持現有風格，即使你有不同偏好
- 發現無關的問題，提出來，不要自行修掉

#### 當你的改動製造了孤兒
- 移除因為**你的改動**而變成 unused 的 import / 變數 / 函數
- 不要動改動前就已存在的 dead code，除非使用者明確要求

#### 自我檢查
每一行改動，都能直接對應到使用者的要求嗎？

---

### 4. 目標導向執行

**定義成功標準，循環驗證直到達成。**

#### 把任務轉換成可驗證的目標

| 原始說法 | 轉換為… |
|----------|---------|
| 「加入驗證」 | 先寫非法輸入的測試，再讓測試通過 |
| 「修這個 bug」 | 先寫能重現 bug 的測試，再修到通過 |
| 「重構 X」 | 確認重構前後測試都通過 |
| 「實作功能 Y」 | 定義驗收標準 → 寫測試 → 實作 → 驗證 |

#### 多步驟任務格式

```
1. [步驟] → 驗證：[確認方式]
2. [步驟] → 驗證：[確認方式]
3. [步驟] → 驗證：[確認方式]
```

#### 堅持規則
先嘗試 3-5 種不同方法再尋求協助。

#### 自我檢查
成功標準是否具體到可以獨立驗證？

---

### 快速參照

| 準則 | 一句話 | 預防的失敗模式 |
|------|--------|----------------|
| 先想清楚，再動手 | 先問，不假設 | 默默做出錯誤假設 |
| 簡單優先 | 少即是多 | 過度設計、程式碼膨脹 |
| 精準修改 | 守住範圍 | 意外的副作用 |
| 目標導向執行 | 測試並循環 | 工作不完整或未驗證 |
